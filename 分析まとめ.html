分析　まとめ

データ分析の４つのプロセス

1. まず大雑把に把握する

まずは、大雑把に全体像を把握して、気になる箇所を詳細に掘り下げる方法がいいでしょう。最初から掘り下げると、出戻りが発生して時間のロスに繋がることがあるので気をつけるようにしましょう。

2. グラフ化する

数字だけみても、どう推移しているのかよくわからないので、視覚的に把握することが大切です。

さらに、データから取得したい情報によって使用するグラフの種類が変わります。

見たい情報 適切なグラフ
推移　　　 折れ線グラフ
割合　　　 円グラフ・積み立て棒グラフ
比較　　　 折れ線グラフ・棒グラフ
分布・相関 散布図・バブルチャート

3. 複数の数値で比較する

売上だけを集計していると、売上の上下しか分かりません。

売上を構成する、購入回数・購入単価・リピートなどの周辺データも一緒にみることで、その数値がでた理由を考えることができるようになります。

例えば、 - 関連する複数の指標を並べて比較する - データを細分化して確認する - 先月、昨年のデータと比較する - 同様の指標を他のサービスと比較する - 関連する指標で割って比較する →　売上を購入ユーザー数で割ったり、訪問回数をCV数で割ったりして、新たな指標を作成する。

4. 仮説を立てて検証する

大雑把にデータを把握し、複数の指標と比較している際に、さまざまな疑問や仮説が浮かび上がります。ここがポイントなんじゃないか？と仮説を立てて検証していくことが大切です。

売上集計

【目標】現状の把握と、時系列による状態変化の把握

  1. 日別の売上・平均購入額の集計
  2. 移動平均を用いて、日別の推移を見る
  3. 月別の売上・累積売上の集計
  4. 月別の売上の対昨年比を集計
  5. Zチャートで業績の推移を見る
  6. カテゴリ別の売上を計算
  7. ファンチャートで商品の売れ行き推移を見る
  8. ヒストグラムで購入価格帯を集計

ユーザー全体の特徴・傾向分析

【目標】セグメントごとの特徴・傾向をつかむ。ユーザーをどのように分けるのが最適かを発見する。

  1. アクション数の集計
  2. ログインユーザーと非ログインユーザーを分けて集計
  3. 年齢別区分を集計
  4. ユーザーの行動頻度を集計
  5. ベン図でアクションを集計・アソシエーション分析
  6. デシル分析
  7. RFM分析（Recency, Frequency, Monetary）
  8. 継続率と定着率を集計
  9. 時系列による状態変化の把握（登録・リピート・離脱）
  10. 継続とリピートに影響するアクションを集計
  11. MAUを分解（新規・リピート・カムバック）
  12. 成長指数を集計

行動分析

【目標】コンバージョンに結びつく行動を見つける

  1. 日次の訪問者数・訪問回数・PVの集計
  2. ページ毎の1の集計
  3. 流入元別に1及び、CVRを集計
  4. PCとスマホでのユーザー行動の違い
  5. 閲覧開始ページと離脱ページの集計
  6. CVに結びつくページの特定
  7. サイト内のユーザーフローを把握する
  8. 完読率を集計する

指標改善の手順

各指標を向上させたいと思ったときは、達成していない人を達成している人に寄せるということを基本の考えとして持って行きます。

  • 達成させたい行動を決める
  • 達成する「前の行動・特徴」の中から、着目する行動を決める
  • 上記の注目する行動の有無や回数を集計し、その後に達成させたい行動を満たしているユーザーの割合を比較する
より投稿やコメントしてもらいたい

→　フォローされている人数によって違いがあるか
→　フォローしている人数によって違いがあるか
→　プロフィール画像の登録によって違いがあるか
新規ユーザーのリピート率を改善したい

→　登録初月の投稿回数によって違いがあるか
→　登録初月にフォローした人数によって違いがあるか
→　登録翌月から7日間以内の利用日数に違いがあるか

改善したい指標に対して、さまざまな仮説を立てて検証を進めることで、注力すべき箇所や施策が見えてきます。

レポートにするときのポイント

上記の様なアクセス解析は一つ一つを見ると、「点」の情報です。それゆえ、全体像を捉えきれず、十分に生かすことができません。

そのため、「点」の情報を頼りに、施策を打ったとしても、局所最適化に陥るので、効率的ではありません。

点と点を結び「線」として捉え、線と線をむすび「面」として捉える。このように全体像を把握した上で最適化の施策を考えることが重要です。

俯瞰図でサービスの全体像を把握するレポートえお作れるかどうか？それをみんなが理解できるようなレポートにできるか？が分析担当者の腕の見せ所です。

countの注意点

集計でcountを使うときは、値がどうなっているのか？（NUllや0など）をきちんと確認すること。

NULL伝播に気をつける

-- 1ユーザーあたりのPV数
1.0 * count(*) / NULLIF(count(distinct long_session), 0)
  • 1.0を掛ける
  • NULLIFを使って、一日で誰もアクセスしなかった日に０を代入する
  • →　NULL伝播があるから

出口ページ判定

-- 出口ページ判定
when row_number() over (partition by session order by stamp desc) = 1 then 1 else 0 end
  • row_number() over ()で、sessionごとにグループ化し、出口ページに1をつけておく(それ以外は0)
  • そうすると、sum()で、出口カウントが簡単にできる。
  • こういうフラグをつける系の技は頻繁に使用する。

完読率を集計する

完読率を集計するためには、どこまで閲覧されたかを把握するログデータが必要です。

そのために、Javascriptでログをそうしｎする仕組みを用意する仕組みがあります。 あるいは、特定のモジュールが表示された段階でログを送信することもできます。
