5.1 過学習の基礎知識¶

正則化とは､**モデルの複雑さを抑えることで過学習を防ぐテクニック**です｡前回扱った多項式回帰は､それまでの線形回帰と比べてモデルが複雑になりやすいという欠点があり､過学習を起こす可能性が高くなっていました｡そこで､過学習を防ぐ正則化が活躍します｡ところで､正則化の手法には代表的なものが3種類あり､それぞれ Ridge ､ LASSO ､ ElasticNet といいます｡これらの手法はややコンセプトが異なっており､それを使い分けられるようになるところまでが本章の目標です｡

まず初めに､正則化が対処する過学習というもの自体についての理解を深めましょう｡ここを押さえることによって､このあと解説する正則化のコンセプトをより深く理解できるようになります｡

5.1.1 過学習とは（復習）¶

重回帰分析の章で触れましたが､過学習とは学習データには正確な予測ができるけど､将来のデータに対してはそれほどでもないというモデルの状態のことです｡trainデータにしかない､本質的でない特徴に過剰に適合してしまった結果､汎化性能を損なってしまっています｡これでは､将来の予測をさせる上では信頼できません｡

5.1.2 誤差を生む要因¶

過学習をしてしまったモデルは未知のサンプルへの予測誤差が大きいわけですが､このような予測誤差を生む要因は大きく3つに分けられます｡それが､バイアスとバリアンスとノイズであり､モデルの誤差に関して以下の方程式が成り立ちます｡これを踏まえると､正則化の役割はバリアンスを抑えることで､バイアスとバリアンスのバランスを調整することだといえます｡

バイアス (Bias)¶

バイアスとは､モデルの表現力不足によって生じる誤差です｡たとえば､3次関数に従ったデータを線形回帰したときはバイアスが大きい（未学習）といえます｡表現力の低いモデルを利用しているために､意味のあるパターンを拾いきれていないのです｡

バリアンス (Variance)¶

バリアンスとは､trainデータに過剰適合することによって生じる誤差です｡たとえば､3次関数に従ったデータを7次関数で多項式回帰したときはバリアンスが大きい（過学習）といえます｡表現力が高すぎるモデルを利用しているために､trainデータにしかないパターンを拾ってしまっているのです｡

ノイズ (Noise)¶

ノイズとは､データに混入している本質的でない情報です｡ノイズは本質的な情報と混ざってデータ中に存在しており､両者を区別することができません｡そのため､機械学習の手法を用いても取り除くことができません｡

5.1.3 過学習の原因¶

バリアンスを高くして､過学習を引き起こしてしまう原因を2つ紹介します｡これらを防ぐことで､バリアンスを抑えることができます｡

原因1. 重みパラメータの値が極端になっている¶

原因のひとつめは､重みパラメータの絶対値が極端に大きくなっていることです｡なぜなら､極端なパラメータはモデルを複雑な関数にしてしまうためです｡例えば､多項式の場合であれば､グラフをぐにゃぐにゃに曲げてしまいます｡なので､重みパラメータの値を平易に保つことができればモデルが平易でなめらかな形になり､バリアンスは抑えられます｡

原因2. サンプル数に対して説明変数が多すぎる¶

また､説明変数が多すぎることも過学習を引き起こす原因のひとつです｡説明変数が多くなると､調整すべきパラメータの数が増えてしまったり､多重共線性の可能性も高まるために過学習が起こりやすくなります｡一般に､学習すべきパラメータが増えると､学習により多くのサンプルを必要とするようになります｡また､線形モデルの場合はパラメータの数は説明変数の数にほぼ一致するので､説明変数を減らすことで改善されます｡多項式の場合は次数を落としてもパラメータを減らせます｡なので､説明変数を減らせば､あるいは多項式の次数を落とせばバリアンスは抑えられます｡

5.2 正則化のアルゴリズム¶

正則化のアルゴリズムは､上記の2つの原因への対応策になっています｡重みパラメータの値を平易なものにし､手法によっては説明変数を自動的に減らしてもくれます｡まずは､あらゆる正則化手法に共通する､重みパラメータへの制限について解説します｡

5.2.1 重みベクトル $\boldsymbol{w}$ の大きさ制限¶

正則化手法の共通方針は､重みベクトル $\boldsymbol{w}$ の大きさ制限することです｡こうすることによって､その各要素である重みパラメータの値は平易に保たれます｡まずは､大きさの制限がどのような意味を持つのかを視覚的に理解するところから始めましょう｡

正則化なしの勾配降下法¶

まずは､正則化なしの勾配降下法のイメージを確認します｡ここでは､可視化の都合上パラメータが2つの場合を考えます｡この場合は､二乗誤差関数は以下のような立体になります｡勾配降下法を用いてこの立体の勾配を下り､最適化を試みます｡

ところで､この立体を真上から見ると以下の図のような平面になります｡ちなみに､図中の輪は二乗誤差関数の等高線です｡ここで意識したいことは､最適なパラメータを学習させたとき､そのパラメータの組み合わせが二乗誤差関数の極小値となる点を示す位置ベクトルになっているということです｡勾配降下法による最適化は､最適な位置ベクトルを定めることを表します｡

正則化による最適化¶

正則化を行う場合は､解がとある原点中心の図形上に存在するように制限がかかります｡あとの節で詳しく説明しますが､この図形の大きさは重みベクトルの大きさを表しています｡そのため､この図形の大きさを制限することが重みベクトルの大きさへの制限（罰則）になります｡また､このような制限をかけたときは､解は勾配を下って極小値に近づこうとするものの､その図形上を出ることはできません｡そのため､最終的には図形上で最も二乗誤差関数が小さい点に解が落ち着きます｡

5.2.2 正則化の目的関数¶

正則化による最適化は､目的関数に正則化項という項を加えてつくられています｡式中の $\lambda E(\boldsymbol{w})$ の部分がその正則化項です｡$E(\boldsymbol{w})$ の中にはベクトルの大きさを表す式が中に入り､ $\lambda$ はその比例定数です｡正則化による最適化は､これを通常の勾配降下法にかけるだけで実行できます｡また､正則化を加えた最小二乗法は､特に正則化最小二乗法と呼ばれます｡

$$Regularization = \frac{1}{n} \sum^{n}_{i=0} ( \hat{y}_i - y_i)^2 + \lambda E(\boldsymbol{w})$$

正則化項を加えた誤差関数を最小化することは､パラメータに対して二乗誤差関数を小さくしようとする圧力と､解の存在範囲を小さくしようとする圧力の両方をかけることを意味します｡前者は解を二乗誤差関数の極小値に近づける向きに､後者は原点に近づける向きに働きます｡すると､2つの力はせめぎ合いになり､最終的にバランスの取れたところに落ち着くようになります｡

$\lambda$ の役割¶

比例定数である $\lambda$ は､正則化の強さを制御するハイパーパラメータです｡$\lambda$ 大きくすればするほど正則化項が誤差関数中で大きな割合を占めるので､ $E(\boldsymbol{w})$ が最小化の圧力を強く受けるようになります｡逆に､小さくするほど誤差関数中での割合が減り､ $E(\boldsymbol{w})$ が受ける圧力は小さくなります｡また､ハイパーパラメータとは､学習の過程で修正されない､分析者があらかじめ設定しておく定数のことです｡

5.2.3 Lpノルム¶

Lpノルムとは､ベクトルの大きさの一般形です｡以下のような式で表され､ $p$ の値を変えることによって大きさのとり方を変えることができます｡正則化では重みベクトルの大きさに制限をかけるといいましたが､実はその大きさには種類があります｡ここでは､L2ノルムとL1ノルムを中心に解説します｡

$$\|\boldsymbol{w}\|_p = (|w_1|^p + |w_2|^p +...+ |w_d|^p)^{\frac{1}{p}}$$

参考資料: Qiita【機械学習】LPノルムってなんだっけ?

L2ノルム¶

$p = 2$ のときのLpノルムをL2ノルムといいます｡L2ノルムは以下の式で表されるノルムで､私たちにもっとも馴染みのある､いわゆる｢ベクトルの大きさ｣です｡また､L2ノルムに罰則を課す正則化回帰をRidge回帰（L2正則化）といいます｡

$$\|\boldsymbol{w}\|_2 = \sqrt{w_1^2 + w_2^2 +...+ w_d^2}$$

L2ノルムを平面上に表現すると､図のような原点から最適解となる点までののユークリッド距離（いわゆる｢2点間の距離｣）を表します｡この場合､L2ノルムの表す等距離線が円形（一般には超球状）になることにも注意してください｡

L1ノルム¶

$p = 1$ のときのLpノルムをL1ノルムといいます｡L1ノルムは以下の式で表されるノルムで､それぞれのパラメータの絶対値を足し合わせた形になっています｡また､L1ノルムに罰則を課す正則化回帰をLASSO（L1正則化）といいます｡

$$\|\boldsymbol{w}\|_1 = |w_1| + |w_2| +...+ |w_d|$$

L1ノルムを平面上に表現すると､図のような縦横の移動距離の合計なります｡また､L2ノルムの等距離線が円形だったのに対し､L1ノルムの等距離線は正方形（一般には超立方体）になります｡

L1とL2の中間のノルム¶

$1 < p < 2$ のときのLpノルムに罰則を課す正則化回帰をElasticNetと呼びます｡このとき､ノルムはL2とL1の中間の形状である､膨らみを持った正方形をしています｡ $p$ は､特に整数でなければならないというような縛りはなく､小数に設定することもできます｡

5.3 正則化の実装¶

この節からは､正則化の実装を進めていきます｡また､正則化手法は $E(\boldsymbol{w})$ の中身となるLpノルムによって種類が分けられています｡ここでは3種類の代表的な正則化手法を紹介し､先程紹介した3種類のノルムのそれぞれに対応しています｡

5.3.1 正則化なしの多項式回帰¶

まず､比較のために正則化なしの多項式回帰を実行します｡このとき､過学習の様子を観察するために､あえてデータのサンプル数を少なくして､必要以上に高い次元の多項式回帰を実行します｡

サンプルデータの生成¶

まず､サンプルとなるデータとして適当な3次関数に従うデータを生成します｡サンプル数は30と非常に少なくしてあります｡

# 適当な三次関数を出力する関数を定義
def cubic(X):
    y = 0.0001 * (X**3 + X**2 + X + 1)
    return y

X = X.reshape(-1, 1)                      # sklearnで扱えるように､一列のベクトルに変換

# 三次関数をプロットする用のデータを生成

5.4.2 Ridge回帰¶

初めに紹介するのは､L2ノルムに罰則を課すRidge回帰です｡ただし､scikit-learnでは､計算が簡潔になるという理由から､L2ノルムの2乗に罰則を課しています｡よって､scikit-learnによるRidge回帰の目的関数は以下のようになります｡

$$J(\boldsymbol{w})_{Ridge} = \sum^{n}_{i=0} \| y^{(i)}-\hat{y}^{(i)} \| ^2 + \lambda \|\boldsymbol{w}\|^2_2$$

Ridge回帰の実行¶

Ridge回帰はlinear_modelのRidgeクラスを利用して実行できます｡alpha（正則化項の係数 $\lambda$ のこと）の値を変えながら繰り返し実行してみましょう｡回帰係数や決定係数の出力はLinearRegressionと同様に.coef_や.scoreで出力できます｡

5.4.3 LASSO¶

次に､L1ノルムに罰則を課すLASSOです｡LASSOの目的関数が以下のようになります｡

$$J(\boldsymbol{w})_{LASSO} = \frac{1}{2n} \sum^{n}_{i=0} (y^{(i)}-\hat{y}^{(i)})^2 + \lambda\sum^{m}_{j=1} |w_j|$$

自動的な特徴選択¶

特徴選択とは､学習に必要な特徴量を選別して利用することです｡また､利用する特徴量を減らすこと全般を次元削減と言います｡LASSOの最大の特徴は､特徴選択による次元削減を自動的に行えることです｡そもそも､LASSOというのが"Least Absolute Shrinkage and Selection Operator（選択的な縮小推定器）"の略なのです｡｢縮小推定器｣はパラメータ値による影響を小さくしようとする正則化のを､｢選択的な｣はモデルの中で､不要と思われる特徴量を選択して消去するという意味です｡

LASSOに特徴選択ができる理由は､L1ノルムの等高線のとがった形状にあります｡L1ノルムに罰則を課したとき､最適解は等高線の正方形の角に落ち着くことが多くなります｡下図にあるRidge回帰との比較を見ると､それが確認できます｡そして､正方形の角は座標軸上にありますので､このとき何かしらのパラメータの値は0になります｡パラメータが0になるということは､ $w_ix_i = 0 \times x_i = 0$ となり､掛け算されている特徴量が回帰式から消去されることになります｡

これにより､重要でない特徴量は次々に消去されていきます｡このようにすると､最適解となる重みベクトルの要素には0が多くなります｡このような解を疎な解（Sparse Solution）と呼びます｡重要な特徴量だけが生き残り､それらによってのみ予測を行うモデルがつくられるのです｡

LASSOの使い所¶

LASSOの性質が最も活かされるのは､スパースなデータを分析するときです｡スパースなデータでは､サンプル数に対して特徴量が多すぎるという問題が多くつきまといます｡これによって､モデルが過学習をしやすくなってしまいます｡その中には分析にほとんど貢献しない特徴量もあり､LASSOによる特徴選択が有効です｡また､膨大な特徴量に対しての判別が自動的に行われるという点も非常にありがたいものです｡

LASSOの実行¶

LASSOは､linear_modelのLassoクラスを利用して実行できます｡Ridge回帰と同様に､alphaの値を変えながらグラフを観察します｡

5.4.4 Elastic Net¶

最後に､L2とL1の中間のノルムに罰則を課すElasticNetを紹介します｡ElasticNetの目的関数は､以下に示すようなL1ノルムとL2ノルムの両方に同時に罰則を課す構造になっています｡両者の係数 $\lambda_1$ と $\lambda_2$ を調整して両者のバランスをとり､中間のノルムへの罰則を再現します｡

$$J(\boldsymbol{w})_{ElasticNet} = \frac{1}{2n} \sum^{n}_{i=0} ( y^{(i)}-\hat{y}^{(i)} ) ^2 + \lambda_1 \sum^{m}_{j=1} |w_j| + \lambda _2 \|\boldsymbol{w}\|^2 $$

また､scikit-learnではこの式が以下のように変形されています｡一番のポイントは､ハイパーパラメータのとり方が変わったことです｡変形前の式では $\lambda _1$ と $\lambda _2$ を使用していましたが､変形後は $\alpha$ と $L_1 ratio$ を利用しています｡

$$ElasticNet = \frac{1}{2n} \sum^{n}_{i=0} (y^{(i)}-\hat{y}^{(i)})^2 + \alpha *L_1ratio\sum^{m}_{j=1} |w_j| + \frac{1}{2} \alpha* (1 - L_1ratio) \|\boldsymbol{w}\|^2 $$

置き換えた理由は､ハイパーパラメータを解釈しやすくするためです｡変形後の式では $\alpha$ で正則化全体の強さを､$L_1ratio$ でノルムをどの程度L1に近くあるいはL2に近いものにするかを調整することができます｡このような切り分け方でハイパーパラメータを設定すると､後の微調整がしやすくなるのです｡また､ $L_1 ratio$ は比率ですので､0~1の範囲に値を収める必要があります｡

次元削減の強さを微調整¶

ElasticNetは､次元削減の強さを微調整するために設計されました｡L2ノルムとL1ノルムの中間のノルムに罰則を課すと､次元削減の効果もRidge回帰とLASSOの中間の強さになります｡Ridge回帰より強いがLASSOより弱い次元削減が行われるため､次元削減をさせたいがLASSOでは効果が強すぎるという場合に役立つのです｡ $L_1 ratio$ を微調整することで､次元削減の強さを微調整することができます｡
