7.1 確率が得られることの利点¶

ロジスティック回帰は､判定確率を求めることが出来る線形分類器です｡ただ分類をするだけではなく､分類の確信度合いを0~1の確率で教えてくれます｡このような特徴から､臨床医療やダイレクトマーケティングを始めとする広い産業で長く愛されています｡また､最適化の手法にも特徴があり､最尤推定というパーセプトロンよりも柔軟に最適解を求められる手法を採用しています｡

分析ストーリー : 癌の発症確率の算出¶

今回のストーリーは､ロジスティック回帰が特に好まれる臨床医学における利用例を取り上げます｡とある大学病院の臨床科学センターでは､乳がんの直接診断のための細胞診断が導入されていました｡腫瘍が認められた部位から採取した細胞核を光学顕微鏡で観察し､大きさや形､染色時の色合いなどの情報をもとに､乳がんの発症確率を予測することにしました｡

しかし､この検査結果の確認は､医師が逐一肉眼で判定していたのでは､時間がかかりすぎてしまいます｡そういった経緯から､これまでは目視で行ってきた診断を機械学習を利用して自動化するこになりました｡これによって､より多くの患者さんが手軽に診療を受けられる体制をつくることを目指します｡

ロジスティック回帰は､予測確率を出力できるところがパーセプトロンに対する優位性のひとつです｡ここでは､その｢確率｣の表す意味合いや､それが得られることの利点を整理します｡

7.1.1 確率とは¶

確率とは､将来にとあるできごとが起こる確からしさを表す指標です｡起こりうることそれぞれについて0~1の値で表現され､考えられるすべての場合について足し合わせると合計が1になるという性質があります｡私たちに最も身近な例として､天気予報で報じられる一日の降水確率があります｡天気予報では､｢雨が降るかどうか｣という0か1かの判断だけではなく､｢どの程度雨が降ることが見込まれるか｣という程度を表す情報を報じているのです｡

7.1.2 リスクを把握する¶

確率がわかることの最大の利点は､リスクを把握できるようになることです｡たとえば､雨が降るかどうかによって会社に傘を持っていくかどうかを決めたい場合と､家族で遠出をするかどうかを決めたい場合では､判断の基準が変わるかもしれません｡｢明日は雨が降りません｣という結論が出ただけでは､降水確率0%の｢絶対に降らない｣なのか､降水確率49%の｢かなり怪しいけど､どちらかと言えば降らない｣なのかはわからないのです｡

7.2 ロジスティック関数¶

ロジスティック関数（別名: シグモイド関数）とは､実数全体の値をとる入力を､0〜1の値に変換する関数です｡入力に線形入力をとって､出力の数値を確率とみなすことで利用できます｡こちらは､ロジスティック回帰の名前の由来にもなっています｡ここでは､ロジスティック関数の成り立ちを追って､なぜロジスティック関数で確率を表せるのかを学びましょう｡

7.2.1 オッズ（オッズ比）¶

まずは､オッズについて解説します｡オッズ（オッズ比）とは､とあるできごとが起こる確率と起こらない確率の比です｡オッズのとる値の範囲は $0 < odds < +\infty$ であり､確率のとる値の範囲を広げています｡

7.2.2 対数オッズ（ロジット関数）¶

また､オッズの対数を取ったものを対数オッズ､あるいはロジット関数といいます｡対数の底は $e$ (ネイピア数)であるとします｡対数オッズのとる値の範囲は $- \infty < logit(odds) < +\infty$ であり､$0 < p < 1$ の範囲の値を取る確率を実数全体にまで広げることが出来ました｡

7.2.3 ロジスティック関数（シグモイド関数）¶

ロジスティック関数とは､ロジット関数の逆関数です｡ロジット関数とは逆に､実数全体を $0 < p < 1$ の範囲に縮約できます｡このとき､入力である $logit$ を線形入力で置き換えて､2値分類における一方のクラスへの予測確率と捉えることで､予測確率を求めることが可能になります｡

7.3 最尤推定¶

最尤推定とは､ロジスティック回帰の最適化の方法であり､最も尤も（もっとも）らしいパラメータを獲得するというコンセプトで名前がつけられています｡ロジスティック回帰も､線形分類器という面ではパーセプトロンと同じでしたが､この最適化手法の部分は大きく異なります｡最尤推定は誤差のとり方を｢正解したか､してないか｣という画一的な情報ではなく｢正解に対してどれだけ確信を持てたか｣という幅のある情報に変えることができ､パーセプトロンよりも柔軟に最適解に近づくことができます｡

7.3.1 尤度関数¶

尤度関数とは､その名の通り尤もらしさを表す関数であり､以下の式で表現されます｡ロジスティック回帰においては､この尤度関数を最大化するようなパラメータを求めることを目指します｡見た目はなんとも難しそうな印象ですが､要は｢正解に対してどれだけ確信を持てたか｣ということを定量化しているに過ぎません｡この節では､尤度関数の構造を部分ごとに理解していきましょう｡

条件付き確率¶

条件付き確率とは､｢とある『前提条件』があったときの､とある『事象』が起こる確率｣を表しています｡たとえば $P(y^{(i)}=1|\boldsymbol{w} ; \boldsymbol{x})$ は､｢現状の $\boldsymbol{w} ; \boldsymbol{x}$ で $i$ 番目のサンプルを $y = 1$ であると予測する確率｣を表すことになります｡

べき乗の意味¶

尤度関数の $\prod$ 記号の中では､条件付き確率が2つ掛け算され､さらに指数に $y$ や $1-y$ が付いています｡ポイントは､正解ラベルである $y$ は $0$ か $1$ の値をとるため､ $y$ と $1 - y$ は必ず一方が $1$ になり､他方が $0$ になるということです｡指数計算において､ $1$ 乗は｢何の変化もしない｣､ $0$ 乗は｢いかなる数値でも $1$ になる｣の意味を持ちます｡すると､条件付き確率は $i$ 番目のサンプルの正解ラベルに割り当てられた確率だけが残り､不正解のラベルに割り当てられた確率は $\times 1$ になって式中から消えるようになります｡

記号 $\prod$¶

ところで､ $\prod$ の記号は $\sum$ の掛け算版です｡｢すべての値を足し算する｣の意味を持つ $\sum$ に対して､$\prod$ は｢すべての値を掛け算する｣の意味を持っています｡こうすることで､trainデータのサンプル全体で､正解ラベルに割り当てた確率を集計することが出来ます｡こうすることで､データ全体を評価する事ができます｡

7.3.2 対数尤度関数¶

対数尤度関数とは､その名の通り尤度関数の対数をとったものです｡対数をとったのは､確率の｢掛け算｣を｢足し算｣に置き換えるためです｡なぜなら､確率は0~1の値をとるため､掛け算を重ねるに連れて指数関数的に値が小さくなってしまうという欠点があります｡特に､サンプル数が多くなる場合は､尤度関数は常にほとんどゼロになってしまいます｡これでは､学習がされているのかどうかが尤度関数からほとんどわからなくなってしまいます｡

$$logL(\boldsymbol{w}) = \sum^{n}_{i=0} \biggl( y^{(i)} logp_i + (1 - y^{(i)}) log (1 - p_i) \biggr)$$

しかし､対数をとることでこの掛け算が足し算に変わり､掛け算で起こる問題が解決されます｡ちなみに､ロジスティック回帰の目的関数は対数尤度関数に $-1$ を掛け算した $logL(\boldsymbol{w})$ が利用されています｡これは､目的関数を勾配降下法による最小化問題に持ち込むためです｡

7.3.3 最尤推定における誤差関数¶

ロジスティック回帰モデルの学習は､先程定義した誤差関数に勾配降下法を適用すれば実行できます｡ただ､ロジスティック関数はその際に表れる導関数の形が特徴的なので､そこに触れておきます｡実はロジスティック関数には特殊な性質があり､微分を行うとロジスティック関数の形が再び出てきます｡

$$\frac{d}{du} \sigma(u) = \sigma(u) \biggl( 1 - \sigma(u) \biggr)$$

この性質を利用すると､誤差関数の微分もシンプルな形になります｡微分を関数値を代入するだけで算出できるので､計算が簡単になります｡ロジスティック回帰が古くから好まれてきたのは､コンピュータがなくても計算がしやすかったからという理由もあります｡

$$\frac{\partial}{\partial w_j} J(\boldsymbol{w}) = - \sum^{n}_{i=1}\bigl(y^{(i)} -\Phi(z^{(i)})\bigr)x_j^{(i)}$$

7.3.4 逆正則化パラメータ¶

ロジスティック回帰には､デフォルトでL2正則化の機能が組み込まれています｡L2ノルムの正則化項を加えた誤差関数は以下のようになります｡

$$J(w) = - log L(\boldsymbol{w}) + \frac{1}{2} \lambda ||\boldsymbol{w}||^2$$

ただし､ロジスティック回帰の場合は慣習的に逆正則化パラメータという､ $\lambda$ の逆数が用いられます｡右辺を $\lambda$ で割り算したように､対数尤度関数の方に $C$ をかけています｡

$$ C = \frac{1}{\lambda} $$

よって､ロジスティック回帰の目的関数は以下のようになります｡正則化を強くしたい場合､二乗誤差関数の影響を相対的に小さくすれば良いので､ $C$ を小さく設定します｡

$$J(w) = - C log L(\boldsymbol{w}) + \frac{1}{2} ||\boldsymbol{w}||^2$$

# ２次元データの決定曲線をプロットする関数
def plot_decision_regions(X, y, classifier, resolution=0.02):
    from matplotlib.colors import ListedColormap
    markers = ("s", "x", "o", "^", "v")
    colors = ("red", "blue", "lightgreen", "gray", "cyan")
    cmap = ListedColormap(colors[:len(np.unique(y))])
    
    x1_min, x1_max = X[:, 0].min()-1 , X[:, 0].max() +1
    x2_min, x2_max = X[:, 1].min()-1 , X[:, 1].max() +1
    
    xx1, xx2 = np.meshgrid(np.arange(x1_min, x1_max, resolution), np.arange(x2_min, x2_max, resolution))
    Z = classifier.predict(np.array([xx1.ravel(), xx2.ravel()]).T)
    Z = Z.reshape(xx1.shape)
    plt.contourf(xx1, xx2, Z, alpha=0.4, cmap=cmap)
    
    plt.xlim(xx1.min(), xx1.max())
    plt.ylim(xx2.min(), xx2.max())
    
    for idx, cl in enumerate(np.unique(y)):
        plt.scatter(x=X[y == cl, 0], y=X[y == cl, 1], alpha=0.8, c=cmap(idx),marker=markers[idx], label=cl)
