3.1 今回のストーリー¶

重回帰分析とは､複数の説明変数を利用した回帰分析のことです｡単回帰分析ではひとつの説明変数のみを予測に利用していましたので､重回帰は単回帰の拡張版と考えることができます｡下図を見てもらうと､よりはっきりと実感が掴めるかと思います｡

同じ線形回帰で考えれば､単回帰分析では2次元の平面内で直線を用いて回帰を行うのに対し､説明変数が2つの重回帰分析では3次元の空間内で平面を用いて回帰を行うようになります｡また､重回帰分析のときの回帰係数を偏回帰係数と呼びます｡もちろん､説明変数はもっと増すやこともできるので､4次元や5次元の空間内で回帰を行うこともあります｡ちなみに､高次元で直線や平面にあたるものは超平面と呼ばれます｡

分析ストーリー: 住宅価格の相場予測¶

米ボストンに､とある不動産仲介の会社がありました｡現在この会社は､自社サイトのトップページに最新のお買い得物件を表示する機能を追加することを検討しています｡この機能を自動化できれば､自社サイトへの人の流入が増えると考えたのです｡そして､そのためには各住宅の相場価格を回帰分析によって見積もることが必要になりました｡例えば､相場価格を算出することができれば､実際価格との比を取って､割高度を求めることができます｡この値が1より大きいほどその物件は割高で､1より小さいほど割安という意味の評価指標になります｡この会社は､自社に貯まっていたボストン市の各区画の住宅価格データを利用して､このシステムを構築することにしました｡

Bostonデータセット¶

sklearn.datasets¶

今回は､scikit-learnに収録されているBoston市郊外の住宅価格データセットを利用します｡scikit-learnには､手元に分析できるデータがない人でも機械学習を試せるよう､サンプルとなる小規模なデータセットが多数収録されています｡sklearn.datasetsからアクセスでき､住宅価格予測の他にも乳がんの判定､アヤメの花の品種分類､手書き数字の画像分類などのデータが収録されています｡

データの概観¶

Bostonデータセットは､住宅価格を重回帰分析で予測するためのデータセットです｡ボストン郊外居住区画ごとに犯罪件数や､空気汚染の度合い､あるいは住民人数における肉体労働者の割合などの13種類の説明変数と､目的変数である住宅価格の中央値が記されています｡こちらのデータは､1978年にHarrisonとRubinfeldによって調査がなされたものです｡サンプルとなる区画数が506区画と非常に少数なので､重回帰分析の練習のためのタスクとして利用されています｡

ご覧になっておわかりかと思いますが､散布図行列とは複数の変数の組み合わせに対して､散布図とヒストグラムを出力したグラフのことです｡右下がりの対角線上にヒストグラムが並び､それ以外の場所には対応する2変数についての散布図が出力されています｡これによって､それぞれの変数の分布と､複数の変数同士の関係性をひと目で把握することができます｡重要なのは､データに表れているパターンから現実の様子を読み取ろうとする姿勢です｡できるだけ多く､ありそうな仮説を立ててみましょう｡

ヒストグラム¶

ヒストグラムからは､それぞれの変数がどのように分布しているのかを知ることができます｡ヒストグラムは多くの場合､左に示す正規分布という釣鐘型の分布をしています｡平均に最も多くのサンプルが集まって､そこから離れるに連れて数が減っていきます｡出力されたヒストグラムがこの正規分布からずれていた場合､それなりの理由があることが考えられます｡

たとえば今回ですと､INDUSのヒストグラムが二山型になっていることが特徴的です｡INDUSは小売業以外の商業利用割合､つまりは多くが工業地帯の割合となっています｡それが二山のヒストグラムを作っているということは､左の山は通常の居住宅区域､右の山が工業地帯を表しているということが考えられます｡

他には､MEDVの一番右側に離れ小島ができており､しかもそこでヒストグラムが打ち切られていることも特徴的です｡もしかすると調査の際に､MEDVについて｢50万ドル以上｣というくくりを設けて､住宅価格を不自然に丸めてしまったのかもしれません｡もしそうだった場合､こうした観測のクセはデータ分析においては言わばノイズ（本質的な意味のない情報）なので､取り除く必要があります｡この先の分析においても警戒する必要があるでしょう｡

散布図¶

散布図からは､各変数同士がどのような関係性にあるのかを知ることができます｡今回も線形モデルでの回帰を行うため､各説明変数と目的変数MEDVとの直線的な関係性の強さを評価していくことになります｡

現状で言えば､直線的な関係性が強いのはLSTATとRMであるように思われます｡また､LSTATとMEDVの関係は曲線的であるようにも見えますが､曲線による回帰は次章の多項式回帰で解説します｡

3.3 前処理¶

さて､データの観察によって分析の方針が固まってきました｡これから学習のステップに移りたいのですが､その前にしなければならない処理がひとつあります｡それが､データの分割です（本来なら､単回帰分析のときでもすべきですが､説明する文量の関係､でこちらに回しました）｡こちらは､過学習という問題に対処するために行われます｡

過学習¶

過学習とは､学習データには詳しくなったけど､将来のデータに対してはそれほどでもないというようなモデルの状態のことを指していいます｡この状態は､たとえるなら大学受験で高得点を取るために､教科書の問題だけを丸暗記してしまった受験生に似ています｡受験生は､大学受験を突破するために意味のある知識や思考を身に着けなければならず､教科書の問題と答えを丸暗記しても試験問題は解けません｡

それと同じように､機械学習でも学習データから再現性のない些末なパターンを学習してしまい､将来得られる初見のサンプルに対しての正確な予測ができなくなってしまうのです｡本来､教師あり学習は将来の予測を正確に行うことを目指してモデルを構築するので､これでは意味がありません｡このような状態を｢汎化性能（将来のサンプルに対する予測精度）が低い｣といいます｡また､過学習はデータが少ないときほど起こりやすくなります｡十分なデータがないと､重要な法則性が浮かび上がってこずに､些末な部分にばかり目が行くようになってしまうのです｡

ホールド･アウト法¶

ホールド・アウト法とは､データを訓練用（trainデータ）と検証用（testデータ）の2つに分割して学習させる方法です｡これによって､ある程度過学習を検出することができます｡訓練用とは｢学習に利用する｣という意味で､検証用とは｢学習には利用せずに､汎化性能を確認するためだけに利用する｣という意味です｡学習に利用しなければ､testデータはモデルにとっては初めて見るサンプルになるので､将来得られるサンプルと同様に見えるのです｡先程の受験勉強の例で言えば､教科書で勉強している受験生に定期的に模試を受けさせることに相当します｡初見の問題を解かせて､受験本番での出来を推し量るのです｡

3.5 モデルの性能評価¶

続いて､学習させたモデルがどの程度信頼できるものなのかを､以下の4つの指標を用いて評価します｡

数値による評価¶

  1. 決定係数
  2. RMSE

グラフによる評価¶

  1. 残差プロット
  2. グラフの出力

1. 決定係数 $R^2$¶

決定係数とは､重回帰モデルの当てはまりの良さを表す指標です｡score()メソッドで出力できます｡ここで言う｢当てはまりの良さ｣とは､得られた超平面が特徴空間内に存在するデータ点にどれだけ沿ったものになっているかという指標です｡当てはまりの良さは予測の精度に直結しますので､決定係数をチェックすることで､モデルの予測性能を数値で評価することができます｡通常､ $0〜1$ の値を取り､$1$ に近いほど当てはまりが良く､$0$ に近いほど当てはまりが悪いということを意味します (全く当てはまりが良くないと､マイナスの値を取ることもあります)｡

定義式¶

決定係数は､以下のような定義式で表されます｡注目しているのは｢ばらつき｣です｡目的変数のばらつきには､平面に平行な成分と､平面に垂直な成分がありますが､それらのうち平面に沿った成分がどれだけあるかに着目します｡平面に垂直な成分が少ないほど､値が大きくなるようにできています｡

評価目安¶

決定係数の評価は､以下の目安に従って行ってください｡

決定係数 評価
0.9以上 非常に当てはまりが良い
0.7~0.9 当てはまりが良い
0.5~0.7 あまり当てはまりが良くない
0.5未満 当てはまりが悪い

※注意
単回帰ではこの決定係数をチェックしませんでしたが､それには理由があります｡実は､式の定義上､単回帰の場合の決定係数は､相関係数の2乗と同じになってしまうのです｡なので､単回帰を行うときは相関係数をチェックすれば決定係数を見る必要はありません｡

自由度調整済み決定係数¶

自由度調整済み決定係数とは､決定係数の改良版です｡実は､決定係数には予測の役に立たない変数を追加していくと､数値が意味もなく大きくなっていってしまうという欠点があり､それを改善しています｡scikit-learnには収録されていませんので､簡単に関数を定義しました｡特別な理由がない限りは､こちらを参考にする方が望ましいです｡評価の目安は決定係数と同様です｡

In [27]:
# 自由度調整済み決定係数
# (決定係数, trainまたはtestのサンプル数, 利用した特徴量の数)
def adjusted(score, n_sample, n_features):
    adjusted_score = 1 - (1 - score) * ((n_sample - 1) / (n_sample - n_features - 1))
    return adjusted_score

2. RMSE（Root Mean Square Error: 平均二乗誤差の平方根）¶

RMSEとは､そのモデルについて､通常､予測が正解からどれだけズレるかを表した指標です｡RMSEは､基本統計量の標準偏差に近いイメージで捉えることができます｡標準偏差がサンプルの平均に対する標準的なズレを表しているのに対して､RMSEは正解に対する予測の標準的なズレを表しています｡

定義式¶

RMSEは､以下の式で表されます｡最小二乗法のときに登場した二乗誤差の平均を取り（MSE）､平方根を取ることで二乗をキャンセルしています｡これによって､｢千ドル$^2$｣単位で表されていた全体的な誤差を｢千ドル｣単位に戻しています｡

$$MSE = \frac{1}{n} \sum^{n}_{i=0} ( \hat{y}^{(i)}-{y}^{(i)} )^2$$

$$RMSE = \sqrt{MSE}$$

残差プロット（Residuals Plot）¶

残差プロットとは､予測の外れ方の傾向を視覚的に掴む手法です｡横軸に予測値､縦軸に残差が取ってあります｡また､残差の情報からモデルの状態を考察することを残差分析と言います｡

$$ Residual = \hat{y} - y $$

In [31]:
# 残差プロットを行う関数を定義
def res_plot(y_train, y_train_pred, y_test, y_test_pred):
    # train, testごとに残差を算出
    res_train = y_train_pred - y_train
    res_test  = y_test_pred - y_test
    
    # train, testごとに(予測値, 残差)をプロット
    plt.figure(figsize=(8, 8))
    plt.scatter(y_train_pred, res_train, color='blue', marker='o', label='train', alpha=0.5)
    plt.scatter(y_test_pred, res_test, color='green', marker='s', label='test', alpha=0.5)
    
    plt.xlabel('Predicted Values')                  # x軸のラベルを表示
    plt.ylabel('Residuals')                         # y軸のラベルを表示
    plt.legend(loc='upper left')                    # 凡例を左上に表示
    plt.hlines(y=0, xmin=-10, xmax=50, color='red') # 残差0の線を表示
    plt.xlim([-10, 50])                             # 表示する横軸の範囲を指定
    plt.show()

前提と着眼点¶

残差プロットは､以下の4つの性質を踏まえると､残差0の横線を中心に､下図のようにランダムに分布するはずです｡また､決定係数が高いほどデータ点が全体的に残差0の横線に近づきます｡

残差の性質

  1. 残差の期待値は0
  2. 残差どうしは互いに独立
  3. 残差は正規分布に従って分布する
  4. 残差の分散は等しい

このとき注目すべきは､外れ値の有無と残差の発生傾向です｡

1. 外れ値¶

外れ値とは､本来ありそうな値から大きく外れた値のことです｡発生の背景として､何か特別な理由があることが多いです｡その他のサンプルと一緒くたにして分析すると､そうした特別な理由がノイズ（汲み取るべきでない余計な情報）となり､モデルが多くのサンプルに当てはまるものにならなくなってしまいます｡より良いモデルを作ろうとするなら､外れ値はサンプルから取り除くべきです｡

2. 残差の発生傾向¶

次に注目すべきは､残差の発生傾向です｡残差の生まれ方に本来ないはずの特殊な傾向が表れているときは､やはりそこにも特別な理由があると考えられます｡異常値などが原因で適切な回帰式が得られていなかったり､あるいはデータのとり方のクセであったりというノイズが表れていたりということが､例として挙げられます｡

今回のBostonデータで言えば､右下の直線上に並んだサンプル群がそれにあたります｡こちらは､恐らく散布図行列のときに取り上げた丸められた50万ドル以上の住宅のデータでしょう｡予測値が小さくなるに連れて､誤差が(50, 0)を通る直線に従って大きくなっているので､それがわかります｡こうしたノイズを取り除くことで､モデルの性能は更に上がることが予想されます｡

平面で観察できる利点¶

残差プロットは､多数の変数を用いた重回帰モデルの性質を2次元の平面に落とし込んで観察することが出来ます｡単回帰のときは､学習させた回帰直線をデータ点に重ねることで観察できましたが､重回帰ではそれが難しくなります｡特に､4次元以上になるとデータの分布の可視化がそもそもできません｡しかし､残差だけを取り出す残差プロットなら､データの次元に関係なく2次元の平面に落とし込んで観察できます｡

3.6 標準化重回帰分析¶

標準化重回帰分析とは､重回帰によって各説明変数の予測における重要性を評価する手法です｡今までのように目的変数の構造を定量化するというよりも､それぞれの説明変数がどの程度予測の役に立っているのかを定量化することが目的になります｡これを行うと､各偏回帰係数が予測への貢献度を表す数値（標準偏回帰係数）に変わります｡

偏回帰係数の問題点¶

みなさんは説明変数の重要度を読み取ろうと思ったとき､｢単純に偏回帰係数を比較すればよさそう｣とは思いませんでしたか？偏回帰係数の絶対値が大きいということは､説明変数の変動1あたりの目的変数への影響が大きいことを意味します｡なので､これは一見よさそうに思えます｡しかし､実はそれはちょっと違います｡

ここで注意しなければならないのは､説明変数の数値1あたりの重みに差があるということです｡たとえば､みなさんは｢1円のお金｣と｢1年の時間｣を同じ価値だと思うでしょうか？恐らく､そうは思わないと思います｡なぜなら､お金は生涯年収で3億円ほど手に入ると言われていますが､時間はせいぜい80年あるかどうかだからです｡3億あるうちの1なのか､それとも80しかないうちの1なのかでは､変動1あたりの重みが変わってきます｡通常､重回帰分析に用いる説明変数も単位がバラバラなので､こういったスケーリングの問題が起こります｡これを解消することができれば､回帰係数に各説明変数の重要性だけを表すようにできます｡

標準化¶

標準化とは､変数を標準正規分布のスケーリングに合わせる処理のことです｡一般に､データの分布は何かしらの平均値近くに集まっていて､そこから離れていくにつれて数が減っていくような釣鐘型の正規分布をなすとされています｡このときに､中心位置を表す平均を0に､ばらつきを表す標準偏差を1に統一させることによって､変数のスケールに合わせます｡こうすれば､偏回帰係数が説明変数の重要度だけを表す標準化偏回帰係数になります｡また､その変換は以下の式で実行されます｡

$$z_i = \frac{x_i - \mu}{\sigma}$$

$$x_i: 元データ, \mu: 平均, \sigma: 標準偏差$$

変数の標準化は､sklearn.preprocessingに収録されているStandardScalerクラスを用いて行うことができます｡fit_transform()関数で実行されます｡

In [37]:
# 変数の標準化を実行
from sklearn.preprocessing import StandardScaler
ss = StandardScaler()
X_std = ss.fit_transform(boston.data) # すべての説明変数を変換
y_std = ss.fit_transform(y)

3.7 重回帰分析における注意点¶

重回帰分析について､注意点を3つほどピックアップしました｡単回帰のときにはなかった､説明変数を複数利用するからこその注意点になっています｡こちらを留意の上､分析を行うようにしてください｡

抑制変数¶

抑制変数とは､目的変数との相関が小さくても予測における重要度が大きい説明変数のことです｡今回でいえば､DISがそれに該当します｡実は､重回帰においては相関係数というものは重要度を判断するためのひとつの目安に過ぎず､相関係数だけでは重要度を取りこぼすこともあるのです｡抑制変数の疑いがある変数が見つかったときは､その変数を利用した学習もさせてみて､自由度調整済み決定係数がより大きいモデルを採用するようにしましょう｡

多重共線性¶

多重共線性(multicollinearity)とは､相関が強い説明変数同士を一緒に利用したために､回帰式が信頼できないものになってしまうことをいいます｡たとえば､今回の予測を互いに相関が強い2つの変数を用いて予測すると､以下の図のようになります｡2つの説明変数は相関関係が強いので､データ点は一本の直線上にまとまるように配置されることになります｡すると､ここに置かれる平面が不安定になります｡直線の上に平面を乗せるので､平面がくるくると回転するようになってしまうのです｡これでは､定まった平面が本当にあるべきものになっているのかどうかが疑わしくなります｡

交互作用¶

交互作用とは､とある説明変数の効果が､別の説明変数の値によって変化する作用のことです｡たとえば､とある病気に対する薬の効果が目的変数だったときに､薬の種類のよっては大人にはよく効くけど､子どもにはむしろ悪影響があるということが挙げられます｡｢どの薬を飲むか｣という変数と､｢飲む人が大人か子どもか｣という変数が互いに影響しているのです｡

重回帰で扱う線形モデルは､構造上､交互作用を表現することができません｡言い方を変えれば､交互作用が存在しないという仮定のもとで予測を行っているのです｡交互作用を考慮に含めるためには､この式の構造を変化させる必要があります｡
