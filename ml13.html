13.1 クラスタリングの主な活用例¶

クラスタリングとは､似ているサンプル同士をグループ（クラスタ）にまとめる教師なし学習の手法です｡正解ラベルが用意されていないため､分けられたクラスタには意味付けがされていません｡そのため､意味付けは分析者が結果を見て主観的に行う必要があります｡サンプルをいくつかの組に分けるという点で分類問題とも似ていますが､両者はこの点で異なります｡また､分析者の主観が入るため客観性には乏しく､データの構造にアタリをつけるために利用されることが多い手法でもあります｡

クラスタリングの**｢似ているものどうしをグループにまとめる｣**という機能は非常に応用範囲が広く､ビジネスや学術研究で広く利用されています｡以下には､私たちに身近なビジネス寄りの例を挙げました｡

1. 顧客のセグメンテーション¶

ビジネスの現場では､アンケートなどを利用して**顧客を性別･年齢･職業･世帯人数･嗜好などから細分化する**ことがあります｡たとえば､｢新商品の車を､高級志向で家族連れが多い層に対して作ろう｣などの取り決めをします｡これは**セグメンテーション分析**と呼ばれ､新規事業のターゲットを明確に定義したいときなどに重宝されます｡

2. 商品のレコメンデーション¶

顧客の購買履歴をもとに､**商品の類似性を分析して､顧客が好みそうな商品を推薦する**こともできます｡ECサイトによくある､｢これを買った人は､こんな商品も買っています｣という機能がそれにあたります｡顧客の購買を促すために利用されます｡

3. ニュース記事のトピック分類¶

ニュース記事含まれている単語から､そのニュースのトピックを予測すると言った場合もクラスタリングは役に立ちます｡たとえば､とあるニュースのトピックが｢政治｣なのか､｢スポーツ｣なのか､｢グルメ｣なのかを分けると言った感じです｡これによって､記事の自動タグ付けが行えます｡

13.2 クラスタリングの4つの焦点¶

さて､クラスタリングの実用例について話をしてきましたが､ここからはアルゴリズムの扱い方についての話をしていきます｡**クラスタリングを利用するために､どのようなことを考えていけばいいのでしょうか**？一般的に､クラスタリングのアルゴリズムにおいて焦点となるのは以下の4点です｡これらをどう決めるかで､分析の内容や結果が大きく変わります｡クラスタリングを行おうと思ったときには､初めにチェックしましょう｡

1. 何を分けるのか？¶

まずは､**クラスタリングの対象を明確にする**ところから始まります｡具体的には､顧客などのサンプルを分けるのか､あるいは推薦する商品のような変数を分けるのかに二分されます｡ちなみに､今回はirisのサンプルを対象にクラスタリングを行います｡

2. 階層型か？非階層型か？¶

次に､**クラスタリングの形式**を選択します｡クラスタリングの形式は大きく2種類あり､**単純に対象のグループ分けを行う非階層型のクラスタリング**と､**類似の段階別にグループ分けを行う階層型のクラスタリング**です｡階層型のクラスタリングは図のような**木構造**を作るので､決定木による分類が思い出されます｡両者の違いは､やはり利用するデータが教師ありデータなのか､教師なしデータなのかです｡今回の内容では､非階層型のクラスタリングのみを扱います｡

3. 類似度を何で定義するか？¶

次に重要なのが､グループ分けの基準になる**類似度の定義**です｡**類似性とは性質の近さであり､近さとは特徴空間における距離のことです**｡そこで､**特徴空間内でどのように距離を測るか**を決めます｡直交座標系で使うユークリッド距離､特徴量の相関を考慮したマハラノビス距離､ベクトルの向きで類似性を評価するコサイン距離などがあり､これを使い分けていきます｡ また､今回はユークリッド距離だけを利用します｡

4. どのアルゴリズムを使うか？¶

クラスタリングと一口に言っても､そのアルゴリズムはいくつかあります｡アルゴリズムごとに特性があるので､目的やデータの状態によって使い分けが必要です｡今回は､非階層型クラスタリングからk-means法を紹介します｡

13.3 主成分分析の利用¶

ところで､クラスタリングは**前章で紹介した主成分分析と組み合わせると､都合のいいことがあります｡今回の実装の中でも､主成分分析をクラスタリングの前に適用しています**｡以下では､その都合のいいことを3つほど紹介します｡

1. 次元削減によって､｢次元の呪い｣を回避できる¶

まずひとつは､**次元の呪いの回避**です｡次元の呪いとは､**高次元の空間で起こる､3次元に生きる人間からすると直感に反するような現象の総称**のことです｡クラスタリングでは､その次元の呪いのひとつである､**｢次元を高くするにつれて､データ点同士の距離に差がなくなっていく｣**という現象が起こります｡

この現象は球面集中現象と呼ばれますが､ここでは詳しくは説明しませんので､以下の記事などを参考にしてください｡
蛍光ペンの交差点[別館] ｢次元の呪い､あるいは｢サクサクメロンパン問題｣｣

2. 主成分同士に相関がないので､ユークリッド距離が使える¶

言われてみればそうなのですが､**ユークリッド距離は直交座標空間で定義されるものなので､各特徴量に相関がまったくないという前提のもとで類似度が算出されます**｡しかし､実際は特徴量の間には何かしらの相関関係がある事が多いです｡なので､ユークリッド距離による類似度は正確ではなくなります｡しかし､**主成分分析をすれば各主成分の相関を完全にゼロに出来るので､その心配がなくなります**｡

3. 主要な潜在変数に次元削減できるので､結果を説明しやすい¶

クラスタリングは実際には高次元空間で行うことが多いのですが､**これが2次元まで次元削減できると､急に他者への説明がしやすくなります**｡しかもこのとき､**特徴抽出をすればデータを要約する情報量の多い潜在変数で軸を取れる**ので､内容もより意味深いものになります｡例えば､20代の衣服の購入履歴をクラスタリングにかけた結果をもとに新商品の開発をしたいときには､以下のような図を用いて周囲に説明ができます｡

13.4 k-means法の実装¶

非階層型クラスタリングから､最も有名であるアルゴリズムのk-means法を紹介します｡k-means法は**計算効率がよい**という性質があります｡なので､**大規模なデータの概観を掴む際にも重宝されます**｡

13.4.1 アルゴリズム¶

k-means法のアルゴリズムは以下の通りです｡図を見ながらイメージを掴みましょう｡

  1. クラスタの数 $k$ を指定する
  2. クラスタの中心（セントロイド）の初期値として､ ランダムに選んだ $k$ 個のサンプルの位置を採用する（図②）
  3. 各サンプルが所属するクラスタを､それに一番近いセントロイドと同じだとする（図③）
  4. 所属するサンプルの重心を各クラスタごとに取り､それを各クラスタの新たな中心とする（図④⑤）
  5. 終了条件に達するまで､3と4を繰り返す（図⑥以降）

目的関数¶

k-means法の目的関数は､以下の式で表される**クラスタ内誤差平方和(SSE: Sum of the Squared Error)**と呼ばれるものです｡別名､クラスタ慣性と呼ばれ､ km.inertia_ で呼び出すことができます｡この関数は､**｢各クラスタごとの､各サンプルと中心位置との､ユークリッド距離の二乗和｣**を表しています｡

$$SSE = \sum^{n}_{i=1}\sum^{k}_{j=1} w^{(i, j)} ||\boldsymbol{x^{(i)}} - \boldsymbol{\mu^{(j)}}||^2_2$$

$n$ : サンプルの総数
$k$ : クラスタの総数（事前に指定）
$w^{(i, j)}$ : $i$ 番目のサンプルが､ $j$ 番目のクラスタに属するかどうかを表すフラグ（属していれば $1$ ､属していなければ 0）
$\boldsymbol{x}^{(i)}$ : $i$ 番目のサンプルのデータ点の座標
$\boldsymbol{\mu}^{(j)}$ : $j$ 番目のクラスタの中心位置の座標
$||\boldsymbol{x^{(i)}} - \boldsymbol{\mu^{(j)}}||^2_2$ : ユークリッド距離の二乗和

k-meansの最適化問題は､この **$SSE$ の最小化**です｡実は､ユークリッド距離の二乗和は､クラスタのいびつさを表現する関数になっています｡すると､ $SSE$ の最小化は､**クラスタのいびつさを解消する方向に向かわせる**行為にあたるので､それぞれのクラスタはだんだんと**超球状のきれいに丸まった形**に近づいていきます｡

終了条件¶

k-means法の終了条件には､以下の3つがあり､**どれかひとつに該当した時点で終了**となります｡

  1. クラスタリングが収束して､3と4を行ってもサンプルの遷移がなくなる
  2. $SSE$が許容範囲内にまとまる
  3. 3と4の繰り返しが､指定の回数行われる

# クラスタリングの結果をプロットする関数
# (クラスタの数, 学習器, 学習データ)
def kmeans_plot(n_clusters, km, X):
    # クラスタの予測値を算出
    y_km = km.fit_predict(X)
    
    # クラスタごとに散布図をプロット
    # 5クラスまでプロットできる
    for i, color, marker in zip(range(n_clusters), 'rgbcm', '>o+xv'):
        plt.scatter(X[y_km==i, 0],            # 横軸の値
                    X[y_km==i, 1],            # 縦軸の値
                    color=color,              # プロットの色
                    marker=marker,            # プロットの形
                    label='cluster ' + str(i) # ラベル
                   )
    
    # クラスタの中心をプロット
    plt.scatter(km.cluster_centers_[:, 0],    # 横軸の値
                km.cluster_centers_[:, 1],    # 縦軸の値
                color='y',                    # プロットの色
                marker='*',                   # プロットの形
                label='centroids',            # ラベル
                s=300,                        # プロットのサイズを大きくして見やすくする
               )
    
    plt.legend()
    plt.grid()
    plt.show()

13.5 k-means法の弱点と対策¶

さて､irisデータを用いたクラスタリングは､かなりの精度が出ました｡その理由にはirisデータが単純であるということもありますが､k-means法がクラスタリングを適切に行える強力なアルゴリズムであるということもあります｡さて､そんなk-means法ですが､**実はいくつかの弱点があります**｡k-means法を利用するときには､**その弱点を理解し､対策できるところは対策をしていかなくてはなりません**｡それについて以下で解説します｡

13.5.1 弱点1. 超球状でないクラスタや､サンプル数の異なるクラスタの検出が苦手¶

対策: なし¶

最初の弱点は､**適切に分割できるデータ点の分布に条件がある**ということです｡実は､**k-meansの目的関数である $SSE$ は式の都合上､超球状でサンプル数が同程度のクラスタが生まれることを暗黙の内に仮定しています**｡超球状になりやすい理由は､**ユークリッド距離の二乗を累積するので､距離が大きいことに厳しい罰則がかかる**ことなどであり､クラスタごとのサンプル数がおおよそ等しくなりやすい理由は､**クラスタごとの誤差平方和（SE）が等しく重み付けされていること**などです｡

残念ながら､**これに対策らしい対策はありません**｡あるとしたら､目的関数を変えることくらいですが､基本的にはそれはしません｡今のところ出来るのは､**k-meansに限界があることを知っておくくらいのことだけです**｡

13.5.2 弱点2. クラスタの数 $k$ を自分で決めなければならない¶

k-means法では､**クラスタの数 $k$ をあらかじめ自分で決めておかなければなりません**｡しかし､一般的に､ **$k$ は正確に決められるものではなく､ここまでの内容では分析者の勘に頼っている状態です**｡今回のirisのデータのように分布を可視化することができれば､散布図からおよその見当がつきます｡しかし､高次元で行うときはそうは行きません｡というより､可視化できたとしても､下の散布図を眺めるとどれもそれっぽそうな気がします｡何か**ある程度客観的に評価する方法**はないものでしょうか？

対策①: エルボー法¶

そこで役に立つのが､エルボー法です｡エルボー法とは､**k-means法を $k$ を変えて繰り返し行い､ $SSE$ が｢がくっ｣と低下するような $k$ を見つけて目安とする方法**です｡グラフが決まって人の肘のような形になることから､エルボー法と名付けられています｡

エルボー法は､**｢ $k$ が小さすぎると､複数のクラスタを無理に統合することになるので､ $SSE$ が急に上がるだろう｣**という仮定に基づいています｡この｢がくっと｣というところが重要です｡なぜなら､ **$k$ が増えれば増えるほどセントロイドが多く存在することになり､近くのセントロイドを見つけるのが容易になって $SSE$ が小さくなりやすくなるという､一般的な傾向があるからです**｡今回のケースで言えば､それらしい $k$ の値は2または3といったところでしょうか｡

対策②: シルエット分析¶

もうひとつの対策がシルエット分析です｡ここでは**シルエット係数（silhouette coefficient）**なるものを計算して､クラスタリングの質を判定します｡クラスタリングの質は何で決まるかというと､**｢似たものを似たもの扱いできていること（内部結合）｣**と､**｢似てないものを似てないもの扱いできていること（外部分離）｣**の2つです｡シルエット係数はこれらがどれだけ際立っているかを表す指標になっていて､ます｡

まずは､各サンプル $\boldsymbol{x_i}$ に関して以下の2つの指標を計算します｡

1. 凝集度
データ点 **$\boldsymbol{x_i}$ があるクラスタの､その他のサンプルとの平均距離**｡小さいほど､似た者同士が密にまとまっていて､内部結合が強い｡

2. 乖離度
データ点 **$\boldsymbol{x_i}$ があるクラスタに､一番近い他のクラスタに含まれるサンプルとの平均距離**｡大きいほど､似てないものが遠くにあって､外部分離が強い｡

シルエット係数は､これらを用いて以下のように求められます｡**理論的には､シルエット係数は $-1$ ~ $+1$ の値を取ります**｡

$$シルエット係数 = \frac{乖離度 - 凝集度}{両者のうち大きい方}$$

クラスタ内のサンプルのシルエット係数が大きいほど､､乖離度が凝集度に対して大きいことを示します｡つまりは､**他のものと明確に区別できるクラスタが作られている**ということです｡逆に､**シルエット係数がマイナスになるようなときは､クラスタリングが上手く言っていない**と気づけます｡なぜなら､自分のクラスタ内のサンプルよりも､隣のクラスタ内のサンプルの方が近いということなので､**周囲がごちゃごちゃしている状況の中で無理なクラスタリングが行われてしまっている**ことを示しているからです｡

# シルエット図を出力
from matplotlib import cm

# シルエット図を出力する関数を定義
def draw_silhouettes(X, y_km):
    cluster_labels = np.unique(y_km)                     # クラスラベルを重複なく抽出
    n_clusters = cluster_labels.shape[0]                 # クラスタの数を算出

    # シルエット係数を計算
    # (X, y_km, 距離の種類)
    s = silhouette_samples(X, y_km, metric='euclidean')

    # 各ラベルごとにシルエット図を描画
    y_ax_lower, y_ax_upper= 0, 0                         # シルエット図の上端と下端の初期値を設定
    yticks = []                                          # 縦軸のメモリ位置を格納するリストを生成
    for i, label in enumerate(cluster_labels):
            label_s = s[y_km==label]                     # 該当するクラスタについて､シルエット係数を算出
            label_s.sort()                               # シルエット係数を小さい順に並べ替える
            y_ax_upper += len(label_s)                   # シルエット図の上端を､サンプルの数だけ引き上げる
            color = cm.jet(float(i) / n_clusters)        # color mapから色を取得
            plt.barh(range(y_ax_lower, y_ax_upper),      # 横軸の範囲を指定
                             label_s,                    # バーの幅を指定
                             height=1.0,                 # バーの厚みを指
                             color=color)                # バーの色を指定
            yticks.append((y_ax_lower + y_ax_upper) / 2) # クラスタラベルの表示位置を追加
            y_ax_lower += len(label_s)                   # シルエット図の下端を､サンプルの数だけ引き上げる

            
    plt.axvline(np.mean(s), color="red", linestyle="--") # 係数の平均値に破線を引く(横軸の値, 色, 線の形式)
    plt.yticks(yticks, cluster_labels + 1)               # クラスタレベルを表示(位置, 縦軸の値)
    plt.ylabel('Cluster')
    plt.xlabel('silhouette coefficient')
    plt.show()

# シルエット図を出力
draw_silhouettes(X_pca[:, 0:2], y_km)

シルエット図の見方¶

シルエット図は､以下の2点に注目して**クラスタリングの質**を評価していきます｡そして､**適切な $k$ の値を見つける**ことを目指します｡

チェック項目1. クラスタ内のサンプル数は均等になっているか？¶

シルエット図の縦の厚みは､**クラスタ内のサンプル数**を表していて､ここを見ることによって､**適切なクラスタ数にあたりを付けることができます**｡k-means法は性質上､**正常に収束すると大体同じ数のサンプルが集まったクラスタを検出する**ので､ここにばらつきがあると､**分割しきれていない大きなサンプル群**があったり､**無駄に細かく分けてしまったクラスタがある**ことがわかります｡つまりは､ **$k$ の値が適切ではないのです**｡

チェック項目2. シルエット係数が小さすぎるサンプルはないか？¶

シルエット係数は､**クラスタの内部結合と外部分離の度合いを表す指標**だと言いました｡このことから､シルエット係数が小さいサンプルは､細かくクラスタ分けされていなかったり（内部結合が弱い）､他のクラスタの近くにあって所属が曖昧であったり（外部分離が弱い）します｡**これがマイナスになったりしてしまうと､クラスタリングは上手く言っていないと明確に判断できます｡**｡

クラスタリングが適切でない場合のシルエット図¶

では､ $k$ の値を変えて､クラスタリングが適切でない場合のシルエット図を見ていきましょう｡まずは､ $k$ が小さすぎる場合</font>です｡シルエット係数の値自体は問題なさそうですが､**サンプル数が大きく偏っている**ことがわかります｡こういった場合は､**k-meansが適切にクラスタリングできる状況に反している**ので､ $k$ は不適切だと思われます｡

13.5.3 弱点3. セントロイドの初期値次第で､最終的なクラスタが大きく変わる¶

k-means法はセントロイドの初期値を､ランダムに決定します｡そのときに､**初期値次第では､どう見ても無理のあるクラスタに収束してしまうことがあります**｡このような､初期値によって結果が変わってしまう性質を｢初期値依存性｣といいます｡

こうした収束を抑制するための対策には､以下の2つがあります｡

対策①: k-means法を繰り返し実行する
最も単純な方法は､**k-means法を何回か繰り返し実行して､$SSE$ が一番小さくなった学習器を採用することです**｡そうすることで､**無理のある学習器を採用してしまう確率を減らすことができます**｡ KMeans() の引数である n_init の値を変えることで実行できます｡

対策②: k-means++法を利用する
次に考えられるのは､**セントロイドの初期値のとり方を工夫する**という方法です｡k-means++法はそうした工夫のひとつで､**｢無理のある収束は､セントロイドの初期値が近いときに起こりやすい｣**という傾向に注目して**､離れたサンプル同士で初期値をとるようにしています**｡実行するためには､KMeansの引数の init='random' のところを init='k-means++' とするだけで実行できます｡

というよりも実は､** ```init``` のデフォルト値は ```k-means++``` になっているので､特に指定しなければk-means++法で初期値が設定されます**｡
